## 概要

`stat` はファイルモード（パーミッション）のビット定数と解釈ヘルパーを提供する。HTTP API でラップし、モードの人間可読化と**危険なパーミッション**（world-writable・setuid）の判定を検証した。FT221（tempfile の 0o600）と関連。

| API | ユースケース |
|---|---|
| `stat.S_IMODE(mode)` | パーミッションビット抽出 |
| `stat.filemode(mode)` | `-rw-r--r--` 形式の文字列 |
| `stat.S_IWOTH` / `S_ISUID` | その他書込み権 / setuid ビット |

## 摩擦点

### F-1: `S_IMODE` でパーミッションビットを抽出する

**観察**: `mode` にはファイル種別ビット（S_IFREG 等）とパーミッションビットが混在する。`mode & 0o777` 相当を取り出すには `stat.S_IMODE(mode)`。

**対処**: `S_IMODE` で抽出し `oct()` 表示。グループ/その他の権限から owner-only 判定。

### F-2: `filemode` はファイル種別ビットが要る（`?` 回避）

**観察**: `stat.filemode(0o644)` は先頭が `?`（種別不明）になる。パーミッションのみだと種別が判定できない。

**対処**: 通常ファイル表現として `stat.S_IFREG | perms` を渡し `-rw-r--r--` を得る（実装中に `?` 表示の罠を発見）。

### F-3: 危険なパーミッションの検出（world-writable / setuid）

**観察**: その他書込み権（`S_IWOTH`）は誰でも上書き可能で危険、`S_ISUID`（setuid）は権限昇格の温床。これらを検出できると設定監査に使える。

**対処**: `perms & S_IWOTH` / `perms & S_ISUID` で判定。`0o666`→world-writable、`0o4755`→setuid を検出。
