## 概要

`reprlib` は巨大・深いデータ構造を**省略付き**で `repr` 化する。HTTP API でラップし、文字列長・要素数・ネスト深さの上限で安全に省略表示する用途を検証した。`pprint`（FT235）が整形表示なのに対し、`reprlib` はログ肥大化・出力 DoS 防止に向く。

| API | ユースケース |
|---|---|
| `reprlib.Repr()` | 制限値を持つ repr インスタンス |
| `maxlevel` / `maxstring` / `maxlist` / `maxdict` | 各種上限 |
| `.repr(value)` | 省略付き repr |

## 摩擦点

### F-1: 各上限を個別に設定する必要がある（`maxstring`/`maxlist`/`maxdict`/`maxlevel`）

**観察**: `reprlib.Repr()` は型ごとに別々の上限属性を持つ（文字列は `maxstring`、list は `maxlist`、dict は `maxdict`、ネストは `maxlevel`）。1 つ設定し忘れると、その型だけ無制限で出力が膨らむ。

**対処**: `_build_repr` で関連上限をまとめて設定（`maxother` も含む）。文字列・要素数・深さの 3 軸を API パラメータで制御。

### F-2: 省略は `...` で表現される

**観察**: 上限超過時は `'xxxx...'` や `[0, 1, ...]` のように `...` が挿入される。これを検出すれば「省略が起きたか」を呼び出し側に伝えられる。

**対処**: `"..." in rendered` で `truncated` フラグを返す。長文字列・多要素・深いネストのいずれでも truncated=True を確認。

### F-3: pprint との使い分け

**観察**: `pprint`（FT235）は読みやすい整形が目的で省略は `depth` のみ。`reprlib` は**コンパクトな 1 行 repr + 多軸の上限**が目的でログ向き。

**対処**: 用途で使い分け（人が読む整形=pprint / ログ・短い要約=reprlib）。
