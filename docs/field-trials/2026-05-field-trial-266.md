## 概要

## 摩擦点

### F-1: プロキシ経由の書き込みは `TypeError`

**観察**: `MappingProxyType` への代入（`proxy["k"] = v`）は `TypeError: 'mappingproxy' object does not support item assignment`。設定・定数を公開する際、呼び出し側に変更させない保証になる。

**対処**: 公開する設定は `MappingProxyType` でラップ。診断で `mutation_blocked=True`。

### F-2: ライブビューである（原本変更が反映される）

**観察**: `MappingProxyType` はコピーではなく**ビュー**。原本 dict を変更するとプロキシにも反映される。「スナップショット」ではない点に注意（原本を握る側は変更できる）。

**対処**: 完全な不変が欲しいなら原本も外部に渡さない／`dict(proxy)` でコピーを返す。診断で `reflects_underlying=True`。

### F-3: 浅い読み取り専用（ネストした可変オブジェクトは変更可能）

**観察**: `MappingProxyType` が防ぐのはトップレベルのキー操作のみ。値がリストや dict なら、その**中身は変更できる**（`proxy["items"].append(...)` は通る）。

**対処**: 深い不変が必要なら値も不変化（tuple / frozenset / frozen dataclass）。本 FT はトップレベル読み取り専用の検証に限定。
