## 概要

`colorsys` は RGB / HSV / HLS / YIQ の色空間変換を提供する。HTTP API でラップし RGB ⇄ HSV 変換を検証した。`colorsys` は **0.0〜1.0 の浮動小数**を扱うため、一般的な 0〜255 の RGB との正規化が要点。

| API | ユースケース |
|---|---|
| `colorsys.rgb_to_hsv(r, g, b)` | RGB(0-1) → HSV(0-1) |
| `colorsys.hsv_to_rgb(h, s, v)` | HSV(0-1) → RGB(0-1) |

## 摩擦点

### F-1: colorsys は 0.0〜1.0 を扱う（0〜255 ではない）

**観察**: `colorsys.rgb_to_hsv(255, 0, 0)` は期待どおりに動かない。colorsys は **0.0〜1.0 の正規化値**を前提とする。一般的な 0〜255 の RGB をそのまま渡すと誤った結果になる。

**対処**: 入力 RGB を `/255` で正規化、出力 RGB を `*255` で復元。HSV は 0-1 のまま。純赤 `(255,0,0)`→`hue=0, sat=1, val=1` を確認。

### F-2: 往復精度（丸め）

**観察**: 浮動小数変換で往復すると誤差が出る。出力 RGB は `round()` で整数化。

**対処**: HSV を 4 桁丸め、RGB を四捨五入。`(100,150,200)` の往復で元の値に戻ることを確認。

### F-3: 範囲検証

**観察**: RGB は 0〜255、HSV は 0.0〜1.0 の範囲外だと無意味な結果。

**対処**: Pydantic + 関数内で範囲検証（範囲外は 422）。
