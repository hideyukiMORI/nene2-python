## 概要

`array.array` は型付き（typecode）でメモリ効率の良い数値配列を提供する（`list` より省メモリ）。HTTP API でラップしバイトパックと統計を検証した。typecode の許可リストと範囲外オーバーフロー処理が要点。

| API | ユースケース |
|---|---|
| `array.array(typecode, values)` | 型付き数値配列 |
| `.tobytes()` / `.itemsize` | バイト表現・要素サイズ |

## 摩擦点

### F-1: typecode は許可リスト

**観察**: `array.array(typecode, ...)` の typecode をユーザー入力から取ると、想定外の型（`f`/`d` 浮動小数や `u` Unicode）が混ざる。`z` のような無効値は `ValueError`。

**対処**: 整数 typecode の許可リスト（`b/B/h/H/i/I/l/L/q/Q`）で検証。`z` は 422。

### F-2: 範囲外の値は `OverflowError`

**観察**: `array.array("B", [256])`（`B` は 0-255）は `OverflowError`。typecode の幅を超える値を弾けるのは利点だが、例外処理が必要。

**対処**: `OverflowError` を捕捉して「範囲外」エラーに変換（422）。`B` に 256、`q` に 2^63 が 422。

### F-3: `tobytes()` はネイティブエンディアン

**観察**: `tobytes()` の出力は**実行環境のバイト順**（リトル/ビッグ）に依存する。異なるアーキテクチャ間でバイト列を交換すると解釈がずれる。

**対処**: クロスプラットフォームでバイト列を交換する場合は `struct`（FT215）で明示的にエンディアンを指定する。本 FT はネイティブ前提を明記。
